# Hermes Observer Report Schema

This document defines the HERMES-001A Observer Report fields.

The report is generated by `apps/dockdocs/scripts/generate-observer-report.mjs`.
It is a read-only aggregation artifact. It does not execute queue tasks, merge branches,
deploy production, read secrets, or modify the PMO board.

## Fields

### generatedAt

ISO timestamp for the report generation time.

### currentProductionVersion

Current production baseline from `production-monitoring-baseline.json`.

Required fields:
- `commit`
- `message`
- `productionUrl`
- `netlifyDeployId`
- `evidence`
- `confidence`

### newTasks

Tasks observed as active or newly pending.

Each item includes:
- `id`
- `title`
- `status`
- `evidence`
- `confidence`
- `recommendedAction`

### activeTasks

Current active PMO tasks from the PMO generated board.

### completedTasks

Completed or production tasks from the PMO generated board.

### blockedTasks

Blocked tasks from the PMO generated board.

### productionChanges

Production version, URL, and API endpoint changes observed from the monitoring baseline.

### queueChanges

Generated queue task status changes. Raw command logs are not included.

### queueStatus

Sanitized queue status summary:
- `source`
- `mode`
- `generatedAt`
- `taskCount`
- `pending`
- `running`
- `completed`
- `failed`
- `skipped`
- `containsDangerousCommands`
- `containsChatE2E`

### missionControlHealth

Mission Control generated data health summary:
- `result`
- `source`
- `generatedAt`
- `pmoSyncStatus`
- `queueSource`
- `generatedTaskCount`
- `warningsCount`
- `warnings`
- `evidence`
- `confidence`
- `recommendedAction`

### productionHealth

Production URL and function health from the monitoring baseline.

### knownRisks

Known risks aggregated from production monitoring, generated data warnings, and observer validation warnings.

### recommendedActions

PMO and observer next actions. These are advisory only and do not trigger execution.

## Evidence

Every operational item should include `evidence`, pointing to one or more approved source files:
- `dockdocs-project-board.md`
- `project-board-generated.ts`
- `mission-control-generated-data.ts`
- `codex-task-queue.generated.json`
- `production-monitoring-baseline.json`
- `observer-report.json`

## Confidence

Allowed confidence values:
- `high`: Source is structured and parsed successfully.
- `medium`: Source is present but partially inferred.
- `low`: Source is missing or fallback data was used.

## recommendedAction

`recommendedAction` is a short operational recommendation. It must not include commands that merge, push, deploy, reset, clean, delete files, or expose secrets.
